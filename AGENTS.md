# AGENTS.md

## 架构约束

- 浏览器 DNR API 能实现的功能优先走 DNR 通道；仅在网络层无法实现时使用页面补丁。
- `packages/core` 只放平台无关的规则逻辑，不得依赖浏览器插件全局变量或 React。
- 共享协议、枚举和类型统一维护在 `packages/shared`，不要在应用内重复定义。
- 修改规则协议时，同步检查核心逻辑、插件调用方、持久化数据及迁移逻辑。
- 内容脚本、后台 Service Worker 和 MAIN world 是不同执行上下文；跨上下文通信必须使用类型明确、可序列化的数据。
- 插件运行时代码不得使用仅限 Node.js 的 API，并遵守 MV3 的 Service Worker 与 CSP 限制。

## 开发规范

- 遵循相邻文件现有的 TypeScript、ESM、React 和命名风格。
- 保持模块职责单一，将可复用的规则逻辑从 UI 组件中提取出来。
- UI 改动优先复用现有组件和样式，避免无必要地引入依赖。
- 仅为浏览器限制、跨上下文行为和兼容性处理添加注释。

## 验证要求

- 对所有受影响的 workspace 运行类型检查和最小范围构建。
- 跨包或发布相关改动需运行根目录的 `pnpm typecheck` 与 `pnpm build`。
- 增删依赖或导出时运行 `pnpm knip`。
- 修改插件运行时后，在未打包插件中手动验证相关流程。
- 当前没有根级测试脚本；未实际运行测试时，不得声称自动化测试通过。

## 改动原则

- 只修改任务涉及的代码，不改写无关文件或生成产物。
- 用户可见行为、规则语义或架构发生变化时同步更新文档。
- 交付时列出改动文件、实际执行的验证命令及未完成的检查。
