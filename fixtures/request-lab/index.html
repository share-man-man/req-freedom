LOCAL FIXTURE · REQUEST LAB

Req Freedom 请求实验室

一个零依赖的静态页面，用来观察规则对浏览器请求和页面行为的影响。

页面脚本已就绪
使用方式：使用任意 Node 静态服务托管本目录，打开页面后按卡片中的规则示例配置插件；每次请求的 URL、状态、耗时和响应会显示在下方日志中。Header 改写请同时在 DevTools Network 中查看真实请求/响应头。

NETWORK LAYER · DNR

网络层

作用于全部请求（含页面导航、静态资源），但读不到请求体。

拦截 / 重定向 / 参数注入三选一，Header 改写可叠加。

动态加载小图。命中 block 规则后，卡片会显示资源加载失败，Network 也会显示被阻断。

DNR

重定向

请求 source 文件；重定向后，日志中的最终 URL 和响应内容应变为 target 文件。

/api/redirect-source.json → http://127.0.0.1:4317/api/redirect-target.json
重定向规则示例 
匹配：contains /api/redirect-source.json
目标：http://127.0.0.1:4317/api/redirect-target.json
（须填含协议的绝对地址，端口以当前页面为准）
DNR

参数注入

文件服务会忽略查询参数，但最终请求 URL 会在日志与 Network 面板中清楚显示。

/api/params.json?from=lab
参数规则示例 
匹配：contains /api/params.json
注入：debug=1，environment=local
DNR

请求 / 响应 Header

该请求带有初始 Header；页面会回显可读取的响应头，完整请求头请在 Network 中检查。

/api/headers.json
Header 规则示例 
请求头 set：X-Req-Freedom = enabled
响应头 set：X-Response-Patched = true
COOKIE

Cookie 双向

响应回显收到的 Cookie 请求头，同时下发三条可见性各异的 Set-Cookie。两个方向的改写都能在此核对（首次点击尚无 Cookie 可回显，再点一次即可看到）。

GET /api/cookies
三条 Cookie 的可见性 
lab_visible  Path=/     document.cookie 可见
lab_scoped   Path=/api  仅接口路径回传，页面读不到
lab_session  HttpOnly   JS 一律读不到，只能在
                        Network 面板核对
CROSS ORIGIN

跨域 / CORS

请求另一个端口上的服务（不同源）。「被拦」端点故意不发 CORS 头，浏览器会拦下响应；「对照」端点主动放行，用来证明服务确实可达。

另一端口的跨域服务
解除 CORS 规则示例 
匹配：contains /api/cross-origin/blocked
类型：Header 改写
作用于：响应头
操作：设置
Access-Control-Allow-Origin = *

注意：CORS 由浏览器在网络层裁决，
页面补丁通道看不到被拦下的响应，
只能用 DNR 通道的响应头改写解除。
MULTI METHOD

请求方法

同一个 URL 用不同方法各打一次，响应回显服务端实际处理的方法。用于验证按方法过滤的规则只作用于目标方法。

/api/methods

PAGE PATCH · FETCH / XHR

页面内补丁

仅作用于页面脚本发起的 fetch / XHR，可读写请求体与响应。

返回值 Mock / 限速 / 改请求体可叠加；脚本注入需单独使用（见下方验证区）。

FETCH

基础请求 / 返回值 Mock

读取用户数据。可将这条请求直接 Mock 成任意状态码、响应头和 JSON 响应体。

/api/users.json
Mock 规则示例 
匹配：contains /api/users.json
状态：503
响应体：{"code":503,"message":"mocked"}
XHR

XHR / 返回值 Mock

使用传统 XMLHttpRequest 请求商品数据，专门验证 XHR 的 Mock 与延迟路径。

/api/products.json
Mock 规则示例 
匹配：contains /api/products.json
状态：200

STATUS

任意状态码

按路径返回指定的 HTTP 状态码，用于验证规则在错误响应下的表现，以及 Mock 覆盖真实错误的效果。

/api/status/{code}
FETCH

网络限速 / 延迟

下载约 512 KB 的文本响应。可用 Slow 3G 或自定义延迟观察 loading、耗时和下载过程。

/api/slow-response.txt
限速规则示例 
匹配：contains /api/slow-response.txt
档位：Slow 3G
FETCH POST

上行带宽

向静态路径提交约 16 KB JSON。即使服务端返回 404，也可用来验证请求前的上传等待。

POST /api/upload-probe
FETCH / XHR POST

改请求体

向回显端点提交一段已知 JSON，响应会原样返回服务端实际收到的请求体。对比日志里的 receivedBody 即可确认改写是否生效。

POST /api/echo
改请求体规则示例 
匹配：contains /api/echo
模式：JSON 深合并
内容：{"variables": {"first": 100}}

原始请求体：
{"query":"ListItems","variables":{"first":10,"after":"cursor-0"}}
预期 receivedBody：
{"query":"ListItems","variables":{"first":100,"after":"cursor-0"}}
GRAPHQL

operationName 区分

两个操作共用同一 URL 和方法，只能靠请求体里的 operationName 区分。响应回显服务端实际收到的操作名与变量。

POST /api/graphql
为什么只靠 URL 匹配不够 
ListItems 与 GetUser 的 URL、method 完全相同，
只有请求体不同。仅按 URL 匹配会同时命中两者，
需要「请求体匹配」才能只改其中一个。
