# 隐私政策

生效日期：2026 年 7 月 24 日

Req Freedom 是一款浏览器请求调试扩展。本政策说明扩展为实现请求拦截、重定向、参数与 Header 修改、Mock、延迟模拟、请求体修改和脚本 / CSS 注入功能时，如何处理数据。

## 处理的数据

当用户创建并启用规则后，扩展仅在本地处理与规则匹配的下列数据：

- **网络记录**：当前页面及其请求的 URL、域名和请求方法，用于匹配规则和确定规则的生效范围。
- **用户活动**：页面发起的网络请求活动，用于拦截、重定向、延迟模拟及应用其他已启用的请求规则。
- **网站内容**：命中规则时所需的请求或响应内容、请求头、响应头和请求体，用于实现 Mock、Header 修改、请求体修改及用户配置的动态规则。

这些数据仅在用户浏览器本地、为执行已启用的规则而处理。扩展不会建立浏览历史、请求内容或页面内容的长期记录。

## 本地配置

用户创建的规则、分组、全局开关及导入的配置保存在浏览器的本地扩展存储中。用户可以在扩展内导出、删除或替换这些配置。

## 不收集和不共享的数据

Req Freedom 不会将规则配置、网络记录、请求或响应内容、页面内容或用户活动上传、出售、出租或共享给开发者服务器或第三方。扩展不包含分析、广告或追踪服务，也不使用这些数据进行广告定向、信用评估或贷款用途。

扩展不会专门收集个人身份、健康、财务与付款、身份验证、个人通讯或位置数据。若用户自行配置的规则匹配到可能包含此类信息的请求或响应，相关内容仍只在本地、临时用于执行该规则，不会由扩展传输给开发者或第三方。

## 用户配置的代码

扩展支持用户在规则中填写 JavaScript，用于动态 Mock、动态修改请求体或向匹配页面注入脚本。该代码仅在用户配置的匹配页面中本地执行；Req Freedom 不会下载或执行远程托管的 JavaScript 或 WebAssembly 代码。

用户应只在自己拥有权限的站点和调试环境中使用规则与自定义代码，并自行审查所填写的代码。

## 数据安全与有限使用

Req Freedom 仅将上述数据用于提供和改进其已说明的请求调试功能。不会将用户数据用于与该单一用途无关的目的，也不会出售或转让用户数据。

本扩展对用户数据的使用符合 [Chrome Web Store User Data Policy](https://developer.chrome.com/docs/webstore/program-policies/user-data-faq) 及其 Limited Use 要求。

## 政策更新

如本政策发生实质变化，将在此页面更新生效日期和内容。继续使用扩展即表示你知悉更新后的政策。
