# 导入与导出配置

在规则管理页顶部工具栏可导入或导出全部配置，方便团队共享、备份与迁移浏览器。

## 导出

点击「导出」会下载一个 JSON 文件，包含：

- 全局启用开关
- 分组及其启用状态、顺序
- 每个分组下的全部规则及其字段、顺序
- `schemaVersion` 与导出时间

导出的文件不包含浏览历史、请求内容或其他浏览器数据。

## 导入

点击「导入」并选择此前导出的 JSON 文件。插件会先校验文件结构、执行通道、动作配置、请求方法、枚举值、重复 ID 和正则表达式；校验通过后，确认框会显示即将导入的分组与规则数量。

确认导入会**整体替换**当前的全部分组和全局开关，不会与现有规则合并。建议先导出当前配置作为备份。

## 文件格式

当前格式为 schema 版本 2：

```json
{
  "schemaVersion": 2,
  "exportedAt": "2026-07-19T08:00:00.000Z",
  "enabled": true,
  "groups": []
}
```

规则以 `channel` 区分执行能力：`dnr` 使用浏览器网络层，`page-patch` 使用页面 fetch / XHR 补丁；每条规则在 `actions` 中配置可组合动作，并以 `methods` 限定 HTTP 方法（空数组表示全部）。不支持的 schema 版本不会写入现有配置。
